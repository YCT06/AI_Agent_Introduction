## Slide 1｜開場 — AI Agent 是什麼？

**【開場白】**

大家好，今天這場分享的主題是「AI Agent」。

相信大家對 AI 都不陌生——ChatGPT、Copilot 這類工具很多人已經在用了。但你有沒有想過，現在這些工具，其實只是 AI 能力的「入門階段」？

今天我想帶大家看的，是 AI 正在走向的下一步——從「幫你回答問題」，進化到「幫你完成任務」。這個轉變，就是我們今天要聊的 **AI Agent**。

---

## Slide 2｜情境引入 — 你叫 AI 幫你安排出差

**【情境帶入】**

我先給大家一個具體的情境。

假設你要出差，你打開 AI 說：「幫我安排下週的出差行程。」

如果是現在一般的 AI 助理，它可能會給你一份建議清單：「你可以考慮搭週一早班機、住這間飯店……」然後你還需要自己一項一項去執行。

但如果是 **AI Agent** 呢？它不只是給你建議——

- 它查了你的行事曆，確認你哪幾天有空
- 它自己比價並訂好了機票
- 它預訂了附近的餐廳
- 它甚至通知了所有與會者

你只是告訴它一個目標，剩下的它全部處理掉了。

**這就是 AI Agent 的核心概念：目標告訴它，剩下交給它。**

---

## Slide 3｜AI 的三個演進階段

**【背景脈絡】**

要理解 Agent，我們先來看 AI 這幾年的演進。

AI 大致可以分成三個發展階段：

**第一階段——工具（2022 ChatGPT 時代）**

你問它問題，它給你答案。這是「問答式」的 AI。用途像是 FAQ 回覆、文件起草、企業問答 Bot。人類主導一切，AI 只是一個強大的查詢工具。

**第二階段——助手（現在主流）**

你交代它一件任務，它幫你做好。比如「幫我整理這份會議紀錄」、「幫我生成這週的週報草稿」。AI 開始有了多輪對話、能記住上下文，但每個任務基本上還是需要人來啟動和確認。

**第三階段——代理 Agent（進行中）**

你給它一個目標，它自己規劃、自己執行、自己回報。從「人類下指令 → AI 回應」，進化到「人類設定目標 → AI 自主完成」。

我們現在正處於第二階段往第三階段過渡的關鍵時刻。

---

## Slide 4｜AI Agent 是什麼？

**【核心定義】**

那麼，AI Agent 的正式定義是什麼？

> **代替使用者或系統，自主規劃、判斷、並執行達成目標所需任務的 AI 程式。**

聽起來很抽象，我把它拆成四個核心能力：

**01 理解目標**——解析使用者的最終意圖。不是只聽字面意思，而是理解你真正想要的是什麼。

**02 規劃步驟**——自行決定執行方式與順序。它會自己想：「要達成這個目標，我需要先做 A、再做 B、最後做 C。」

**03 呼叫工具**——使用外部 API 或搜尋引擎。它不是只靠自己的知識，它可以主動去取得資訊、呼叫服務。

**04 反思修正**——根據結果調整下一步行動。做了之後，它會評估：「這個結果對嗎？如果不對，我要怎麼調整？」

這四個能力形成一個「Think → Act → Reflect」的循環，反覆執行，直到目標達成為止。

---

## Slide 5｜Think → Act → Reflect 循環示範

**【循環說明】**

讓我用一個例子來說明這個循環是怎麼運作的。

目標是：**找本週最便宜的台北到東京機票。**

**Think（思考）**：Agent 先分析需求——「我需要搜尋航班資訊，並比較不同日期的票價。」

**Act（行動）**：接著呼叫航班查詢 API，搜尋 7 天內的可用航班。

**Reflect（反思）**：取得結果後評估——「週五最便宜，NT$12,800，這符合需求嗎？」

如果你確認了，它就回報結果並完成任務。如果你說「太貴了」或「我想改日期」，它會**自動繼續循環**，調整條件、重新搜尋，直到找到更好的方案。

這個「想→做→評估→再想」的循環，就是讓 AI Agent 能夠處理複雜任務的關鍵機制。

## Slide 6｜一般 Chatbot vs AI Agent

**【對比說明】**

可能有人會問：「這和我現在用的 ChatGPT 有什麼不同？」

我用這張比較表來說明：

| 比較項目 | 一般 Chatbot | AI Agent |
|---|---|---|
| 互動方式 | 你問、它答 | 給目標、它自己完成 |
| 工具使用 | 無法呼叫外部工具 | 可呼叫外部工具 |
| 跨步驟規劃 | 每次對話獨立 | 可規劃多步驟 |
| 記憶與狀態 | 短暫，多數無記憶 | 可跨步驟保留狀態 |
| 自主性 | 低，依賴人類指令 | 高，自行判斷行動 |

最關鍵的差異是**自主性**。Chatbot 需要你每一步都告訴它做什麼；Agent 則是你給它目標，它自己決定怎麼做到。

一個是「問答機器」，一個是「數位員工」。

## Slide 7｜Agent 的三種類型

**【分類說明】**

Agent 在實際應用上，依照自主程度和架構，可以分成三種類型：

**Workflow 型——按人定義的流程依序執行**

像流水線一樣，人預先設定好所有步驟和分支條件，AI 在每個節點負責局部處理，整體流程不會自行跳脫。舉例來說，「收到客服信件 → 分類 → 自動回覆 → 轉給對應部門」，每個步驟都是人設計好的。這是**目前最普遍的落地方式**，因為行為可預測、容易管控。

**Single Agent 型——AI 自行判斷、呼叫工具、達成目標**

像一個人包辦所有事。接收目標後，AI 自主決定需要哪些工具，逐步執行，並根據結果調整策略。彈性高，但也更難控制。目前**企業開始進入 PoC 驗證階段**，在特定場景下測試可行性。

**Multi-Agent 型——多個 AI 分工協作完成複雜任務**

像一個分工合作的團隊。由多個專責 Agent 組成系統，各自負責子任務，透過協調機制整合輸出。例如一個 Agent 負責研究、一個負責撰寫、一個負責審查。這個方向目前仍在**研究與實驗階段**，但潛力很大。

## Slide 8｜Agentic AI — 下一個更大的概念

**【延伸概念】**

講到這裡，你可能還會聽到一個詞：**Agentic AI**。它跟 AI Agent 有什麼差別？

簡單說：

- **AI Agent**：指的是「一個 AI 個體」，獨立接收目標、自主完成任務。
- **Agentic AI**：指的是「多個 Agent 協同運作的整體系統架構」，分解目標、分工協作、整合輸出。

打個比方：AI Agent 是「一個員工」，Agentic AI 是「一間公司」。

所以 **Agentic AI ≈ Multi-Agent 的概念延伸**——不只是「一個 AI 在動」，而是「一支 AI 團隊在協作」。隨著系統越來越複雜，這個架構層次的思考會越來越重要。

---

## Slide 9｜未來展望

**【發展預測】**

那 AI Agent 的未來會怎麼走？我大致分三個階段來看：

**現在 ～ 2027**：Workflow 型 + 初步 Single Agent 落地。這個階段已經在發生了——許多企業在特定業務場景導入 Workflow 型自動化，少數先行者在測試 Single Agent。

**2028 ～ 2030**：特定業務限定的自律 Agent 實用化。在限定範圍內、有明確規則的任務上，Agent 的自主性會大幅提升，進入真正實用階段。

**2030 年代以後**：通用型完全自律 Agent。這個階段高度不確定，目前還很難預測技術成熟的速度。

有一件事很重要：**全能 AI 不會突然出現。** 它會是「能執行特定任務的 AI 工具」逐步成熟、可靠、普及的過程。我們不需要等待完美的 AGI，現在就可以開始思考哪些任務適合交給 Agent 處理。

## Slide 10｜2025 現狀 — 我們在哪裡？

**【現實評估】**

說了這麼多美好的願景，我們也要面對現實。

**現狀是：多數企業目前仍停留在 Workflow 型，真正自律的 Agent 導入仍是少數。**

為什麼？因為 Agent 技術目前還有很實際的挑戰，分兩個方向來說：

**Single Agent 的挑戰：**
- Hallucination（幻覺）：AI 給出看起來對、但實際上錯的答案
- 推理鏈過淺：遇到複雜問題容易失誤
- 難以驗證中間步驟是否正確

**Multi-Agent 的挑戰：**
- 多個 AI 意圖不同步，任務卡住
- 每次執行結果不穩定
- 內部運作過程不透明，難以除錯

這些都是真實存在的工程問題，不是理論上的假設。

**所以現在最重要的事是什麼？** 不是追求最炫的技術，而是**在現實條件下，找到能穩定落地的 Agent 應用方式**。從小範圍、低風險、可驗證的場景開始，累積經驗與信任。

## Slide 11｜結語 — AI 角色的三個階段

**【總結收尾】**

最後，讓我用一張圖來做個總結。

AI 的角色經歷了三個階段：

- **工具**：問它問題 → 得到答案（ChatGPT 時代）
- **助手**：交代任務 → 它幫你完成（現在主流）
- **代理**：給它目標 → 自己規劃、執行、回報（Agent 時代）

**我們現在正處於從「助手」走向「代理」的過渡期。**

這個過渡期對每一個人、每一間企業來說，都既是挑戰，也是機會。

挑戰在於：技術還不成熟，需要謹慎評估風險。機會在於：現在開始布局、累積 Agent 應用經驗的組織，將在下一個階段佔有先機。

## 附錄｜常見 Q&A

**Q1：AI Agent 和 RPA（機器人流程自動化）有什麼不同？**

RPA 是按照預設規則執行固定流程，遇到例外就會失敗。AI Agent 則能理解語意、自主判斷，遇到例外情況可以自行調整應對策略，彈性大很多。

**Q2：現在有哪些工具可以做 AI Agent？**

常見的框架有 LangChain、LangGraph、AutoGen、CrewAI 等。雲端平台方面，AWS Bedrock Agents、Azure AI Agent Service、Google Vertex AI 都提供託管式的 Agent 服務。

**Q3：導入 AI Agent 的風險是什麼？**

主要風險包括：輸出結果的準確性不穩定、自主操作帶來的安全疑慮（例如 Agent 誤刪資料）、以及成本難以預估（因為每次執行都會呼叫 API）。建議從「低風險 + 有人工審核」的場景開始試行。

**Q4：什麼樣的任務適合交給 AI Agent？**

適合的任務通常具備：步驟明確但繁瑣、需要整合多種資訊來源、重複性高、人工處理耗時但價值相對低。例如：資料蒐集與彙整、初步報告生成、跨系統的資料同步等。
