## 第 3 張｜什麼是 Dify？

Dify 是一個**開源的 LLM 應用開發平台**，它把 Workflow（流程編排）、RAG（知識庫）、Agent（AI 代理）整合在同一個視覺化介面裡。

它的名字來自 **Define + Modify**：
- **Define**：定義你想要的 AI 應用，設定好 Prompt、知識庫、流程
- **Modify**：持續優化，根據實際使用反饋迭代改進

Dify 有幾個核心優勢值得特別提：

**開源授權（Apache 2.0）**，完全免費，可以商用，也可以拿來二次開發。

**可自行部署**，企業的資料不需要上傳到第三方，完全在自己的伺服器上運行，資安風險大幅降低。

**活躍社群**，目前在 GitHub 已有超過 80,000 顆星星，插件生態非常豐富。

GitHub 位置：`github.com/langgenius/dify`

## 第 4 張｜五大核心功能

Dify 整合了 LLM 開發需要的一切工具，可以從原型到生產環境，一站搞定。主要有五大功能：

**① Workflow 視覺化流程**
拖曳式的 LLM 流程編排，支援條件判斷、迴圈、呼叫外部 HTTP API，就像畫流程圖一樣設計你的 AI 邏輯。

**② RAG 知識庫**
上傳 PDF、PPT、TXT 等文件，Dify 自動建立索引，讓 AI 能夠根據你的內部文件回答問題。

**③ Agent 代理**
內建 50 多種工具，包括 Google Search、DALL·E 等，AI 可以自己判斷要呼叫哪個工具來完成任務。

**④ 模型管理**
支援 GPT、Claude、Llama3 等超過一百種 LLM，可以一鍵切換，不綁定單一廠商。

**⑤ LLMOps 監控**
完整的應用監控、日誌分析和 Prompt 版本管理，讓你知道 AI 在線上的運行狀況。

## 第 5 張｜Workflow 視覺化流程設計

Workflow 是 Dify 最核心的功能之一。它的概念很直觀：把 AI 的處理流程分成一個個「節點」，用線條連起來，就像畫一張流程圖。

一個典型的問答流程長這樣：

1. **輸入節點**：接收使用者的問題，可以是文字、上傳的檔案、或 API 傳來的 JSON 資料。
2. **LLM + 工具節點**：這是核心處理區，可以呼叫 LLM 推理、查詢知識庫、執行 Python 程式，或者呼叫外部 API。
3. **輸出節點**：把結果回傳給使用者，或者觸發下一個自動化流程。

**所有節點都可以拖曳連接，完全不需要寫任何程式碼。**

而且它支援複雜邏輯——條件分支（If/Else）、迴圈（Loop）、平行執行——讓你可以設計出非常精細的企業流程自動化。

## 第 7 張｜三種應用類型

根據你的使用場景，Dify 支援三種主要的應用類型：

**① Chatbot 聊天機器人**
這是最常見的應用，適合做客服機器人、知識問答助手。支援多輪對話、記憶上下文，可以整合知識庫即時回答問題。優點是最容易上手，通常是企業導入 AI 的第一步。

**② Workflow 自動化**
適合不需要即時對話、而是需要批次或流程式處理的場景。例如：文件自動摘要、每日報表生成、多個 API 整合串接。這是目前**企業最常用**的 Dify 應用類型。

**③ AI Agent 自主執行**
這是最進階的類型。基於 LLM Function Calling 或 ReAct 框架，AI 可以自行判斷要呼叫哪些工具，完成複雜的多步驟任務。這個類型的彈性最高，但需要更多測試和調整。

## 第 8 張｜RAG 知識庫：讓 AI 讀懂你的文件

很多人問：「我能不能讓 AI 根據我們公司的文件來回答問題？」

答案是可以，這就是 **RAG（Retrieval-Augmented Generation）** 的作用——**檢索增強生成**技術。

運作原理簡單來說：
- 你把公司的文件上傳到 Dify 的知識庫
- 系統自動把文件解析成向量，建立搜尋索引
- 當使用者提問，AI 會先去知識庫裡搜尋最相關的段落
- 再把搜尋到的內容和問題一起送給 LLM，生成準確的回答

這樣的好處是：**AI 回答的內容有所依據，不會亂說**，而且你的文件不需要包含在模型訓練裡，隨時可以更新。

## 第 9 張｜知識庫建構流程

建立一個 Dify 知識庫只需要三個步驟：

**① 上傳文件**
支援幾乎所有常見格式：PDF、Word、PPT、TXT、Markdown。
甚至可以直接連接 Notion、Google Drive 等外部資料來源，讓知識庫自動同步更新。

**② 自動解析與切片**
Dify 會自動提取文字和表格，並使用智慧切片策略（Chunking）把文件拆成適當大小的段落，在保留語意完整性的同時，提升搜尋的精準度。

**③ 嵌入與索引**
文字被向量化（Embedding），儲存到向量資料庫（如 Qdrant、Weaviate）。完成之後，你的 Chatbot 或 Workflow 就可以直接呼叫這個知識庫進行語意搜尋。

**整個過程完全自動，不需要任何程式設計。**

## 第 10 張｜部署方式比較

Dify 提供兩種部署方式，企業可以根據需求選擇：

**☁ 雲端版（dify.ai）**
- 免安裝，直接開始用
- 自動更新，永遠是最新版本
- 有免費方案，付費方案功能無限制
- 適合想快速試用、或沒有 IT 資源的團隊

**🖥 自行部署（Docker / K8S）**
- 資料完全掌控，不外流，符合企業資安要求
- 可以客製化模型與插件
- 支援 Docker Compose 或 Kubernetes 部署
- 適合對資料隱私有嚴格要求的企業

值得一提的是，**開源版本基於 Apache 2.0 授權，完全免費**，即使是商業用途也沒有問題。GitHub 目前已經超過 **80,000 顆星星**，是全球最熱門的 AI 應用開發平台之一。

## 常見 Q&A 參考

**Q：Dify 和 ChatGPT 有什麼不同？**
A：ChatGPT 是一個通用對話工具，而 Dify 是一個讓你**自己打造 AI 應用**的平台。你可以在 Dify 上結合自己公司的文件、流程和工具，打造專屬於你們業務的 AI。

**Q：Dify 支援中文嗎？**
A：完全支援，介面有中文版，知識庫也支援中文文件的解析和搜尋。

**Q：自行部署需要什麼技術能力？**
A：只需要會執行 Docker 指令即可。官方提供的 `docker-compose.yml` 設定檔非常完整，一般後端工程師幾分鐘就能部署完成。

**Q：Dify 的費用是多少？**
A：雲端版有免費方案，付費方案依使用量計費。自行部署的開源版本完全免費，只需要負擔自己的伺服器費用。

**Q：資料安全性如何保障？**
A：選擇自行部署，所有資料都在你自己的伺服器上，不會傳送到 Dify 或任何第三方。這是很多企業選擇開源自架的主要原因。
